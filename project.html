JavaScript Features For Final Exam

1. Fetch API - Real-Time Data Retrieval

The primary JavaScript feature on the Search page is the Fetch API, which retrieves anime data from the Jikan API (MyAnimeList). When users enter a search term and submit the form, JavaScript makes an asynchronous HTTP request to the Jikan API endpoint. The API returns anime data in JSON format, which is then parsed and processed. This demonstrates the ability to work with external APIs, handle asynchronous operations, and manage API responses. The implementation includes error handling to manage network failures or API rate limits.

Location: Search Page

2. Advanced Array Methods - Search & Filter System

The Search page implements a standard filtering system using JavaScript array methods including filter(), map(), and find(). When users select filters by genre, rating, or status, the application uses these array methods to iterate through the fetched anime data and apply conditional logic. Multiple filters can be combined to create complex search queries. This demonstrates understanding of array manipulation, higher-order functions, and conditional logic. The filter system updates search results without requiring additional API calls, improving performance and user experience.

Location: Search Page and Upcoming Anime Page

3. DOM Manipulation - Dynamic Content Rendering

Throughout the website, JavaScript dynamically generates and inserts HTML elements into the DOM using methods like createElement(), appendChild(), and innerHTML. On the Search page, anime cards are created and inserted based on API results. On the Anime Detail page, comprehensive anime information is dynamically rendered. The Watchlist page displays saved anime items. This demonstrates the ability to manipulate the Document Object Model, and update page content in real-time without page reloads.

Location: Search Page, Anime Detail Page, and Watchlist Page

4. Event Listeners - User Interaction Handling

Event listeners are attached to various HTML elements throughout the site to handle user interactions. The Search page uses event listeners on the form submit button, genre filter dropdown, status filter dropdown, and rating input field. The Watchlist page uses event listeners on "Add to Watchlist" buttons and "Remove from Watchlist" buttons. When users interact with these elements, the corresponding event handlers trigger functions like performSearch(), toggleFilter(), addToWatchlist(), and removeFromWatchlist(). This demonstrates understanding of event-driven programming, event delegation, and proper event handler management.

Location: Search Page and Watchlist Page

5. Form Handling & Validation

The Search page implements client-side form validation to ensure users provide valid search input before making API requests. JavaScript prevents default form submission behavior and implements custom validation that checks if the search input is empty. Error messages are dynamically displayed to users if validation fails. The form captures and processes user input from text fields and dropdown selectors. This demonstrates understanding of form handling, input validation, error messaging, and preventDefault() method. Proper validation improves the user experience and reduces unnecessary API calls.

Location: Search Page

Bonus: Local Storage - Data Persistence

The Watchlist page implements browser Local Storage to persist user data across sessions. When users add anime to their watchlist, the data is serialized to JSON format and stored in the browser's localStorage. When the Watchlist page loads, JavaScript retrieves and deserializes the stored data, displaying previously saved items. This feature allows users to maintain their watchlist even after closing their browser.

Location: Watchlist Page

Summary

AnimeHub successfully demonstrates mastery of five essential JavaScript concepts: Fetch API for data retrieval, array methods for data manipulation, DOM manipulation for dynamic content, event listeners for user interaction, and form validation for input handling. The bonus implementation of Local Storage further enhances user experience by providing data persistence. Together, these features create a functional, user-friendly anime discovery platform.
